Znalezione problemy wg priorytetu

P1 @import url() Google Fonts — FOUT (18 komponentów)

Wszystkie komponenty (oprócz yaml-checker, device-health, automation-analyzer) ładują Inter via @import url() w <style> bloku. 8 komponentów ma to PODWÓJNIE (2x @import w dwóch blokach CSS).

Impact: Każdy render (nawet throttlowany) powoduje Flash of Unstyled Text. Na wolniejszym połączeniu widoczny jako mignięcie całego UI.

Fix: Zamienić @import na <link rel="stylesheet"> w renderze HTML (nie w <style>). Link ładuje się raz i cache'uje. Alternatywnie: załadować Inter globalnie w ha-tools-loader-v3.js.

Dotyczy: ha-tools-panel (2x), ha-baby-tracker (2x), ha-cry-analyzer (2x), ha-chore-tracker (2x), ha-backup-manager (2x), ha-security-check (2x), ha-vacuum-water-monitor (2x), ha-storage-monitor (2x), ha-trace-viewer, ha-energy-optimizer, ha-energy-insights, ha-data-exporter, ha-energy-email, ha-log-email, ha-smart-reports, ha-network-map, ha-sentence-manager, ha-purge-cache

P2 Renderowanie WSZYSTKICH tabów (5 komponentów)

Te komponenty budują DOM dla wszystkich tabów i chowają nieaktywne via CSS display:none. To zbędny DOM bloat.

  • ha-device-health: 4 taby (devices, batteries, network, alerts) — wszystkie renderowane
  • ha-baby-tracker: 4 taby (feeding, diapers, sleep, growth) — wszystkie renderowane
  • ha-automation-analyzer: 3 taby (overview, performance, optimization) — wszystkie renderowane
  • ha-chore-tracker: 3 taby (board, schedule, stats) — wszystkie renderowane
  • ha-trace-viewer: 5 detail tabs (summary, flow, timeline, related, json) — wszystkie renderowane

Fix: Renderować tylko aktywny tab (switch/ternary pattern jak w ha-yaml-checker, ha-cry-analyzer, ha-sentence-manager).

P2 Wysoki event rebinding po każdym render (4 komponenty)

Po każdym shadowRoot.innerHTML = stare event listenery są usuwane (DOM replaced), ale nowe są bindowane via querySelectorAll().forEach(addEventListener). Przy dużej ilości listenerów spowalnia to re-render.

  • ha-trace-viewer: ~27 listenerów
  • ha-sentence-manager: ~24 listenerów
  • ha-device-health: ~13 listenerów
  • ha-energy-email: ~13 listenerów

Fix: Event delegation na kontenerze zamiast indywidualnych listenerów. Jeden listener na kontenerze z e.target.closest('[data-action]').

P3 Podwójne bloki CSS / podwójny innerHTML (3 komponenty)

  • ha-vacuum-water-monitor: 2x shadowRoot.innerHTML = (render + updateUI) — może powodować double flash
  • 8 komponentów ma 2 osobne bloki CSS z osobnym @import — font ładowany 2x
Rekomendacje napraw (kolejność)

1. Globalne ładowanie Inter w loaderze (naprawia P1 dla wszystkich)

Dodać <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap"> do <head> strony HA via ha-tools-loader-v3.js. Następnie usunąć wszystkie @import url() z 18 komponentów. Jeden commit, naprawia FOUT globalnie.

2. Active-only tab rendering (naprawia P2 dla 5 komponentów)

Zamienić pattern CSS-hidden tabs na switch/ternary: ha-device-health, ha-baby-tracker, ha-automation-analyzer, ha-chore-tracker, ha-trace-viewer (detail tabs).

3. Event delegation (naprawia P2 dla 4 komponentów)

Zamienić querySelectorAll+forEach na jeden delegowany listener na kontenerze. Priorytet: ha-trace-viewer (27), ha-sentence-manager (24).

  • ha-sentence-manager (po naprawie): hass skip, active-only tabs, <link> fonts
  • ha-trace-viewer: Safe hass setter (firstLoad only)
