**3. DARK MODE**

- Overrides Bento tokens for background, card, text, border, shadows.

- **Result:** Proper dark mode support with Bento token overrides and dynamic canvas color adjustments.

  - `@media (max-width: 768px)` at line ~570
  - `@media (max-width: 480px)` at line ~580
  - `@media (max-width: 360px)` at line ~590

**8. ACCESSIBILITY**

- Color contrast:

  - Bento tokens use accessible colors with sufficient contrast (e.g. `--bento-text` vs `--bento-bg`).
  - Dark mode overrides maintain contrast.
  - Buttons and active states use blue with white text for good contrast.
  - Warning and error colors used with light backgrounds for badges and borders.
  - Chart colors use distinct blues and greens; however, no explicit accessibility features for colorblind users in charts.
- ARIA labels:

  - No explicit `aria-label` or `role` attributes on interactive elements like buttons or tabs.
  - Tabs are buttons with `data-tab` attributes but no ARIA roles or `aria-selected`.
  - Time range buttons and compare mode buttons lack ARIA attributes.
  - Settings overlay is a div with class `.settings-overlay` but no `role="dialog"` or `aria-modal`.
  - Keyboard accessibility is not explicitly handled (e.g. no focus outlines or keyboard navigation hints).
- **Result:** Minor accessibility issues:

  - Missing ARIA roles and labels on tabs and modal dialogs.
  - Potentially insufficient keyboard accessibility support.
  - Color contrast appears acceptable.
