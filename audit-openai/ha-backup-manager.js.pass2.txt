Audit of ha-backup-manager.js CSS and visual consistency:

---

### 1. CARD VS CONTAINER

- The main wrapper uses `<div class="card">` (line ~1020 in _updateUI() method template literal).
- No `.container` class usage found anywhere in the CSS or HTML templates.
- Multiple `.card` and `.backup-card` classes used consistently in CSS.
- Conclusion: Uses `.card` wrapper correctly per Bento design system.

**Count:**
- `.card` wrappers: 1 main card container + multiple `.card` references in CSS.
- `.container`: 0 occurrences.

---

### 2. BENTO TOKENS

- The CSS extensively uses `--bento-*` CSS variables for colors, borders, backgrounds, shadows, radii, transitions, fonts.
- Examples: `var(--bento-primary)`, `var(--bento-bg)`, `var(--bento-text-secondary)`, etc.
- Hardcoded colors found (not using Bento tokens):

  - `#3B82F6` (primary blue) in `.health-value.good` and chart bar color (line ~1040 in CSS).
  - `#10B981` (green) in `.backup-type.full` background rgba and color.
  - `#F59E0B` (orange) in `.backup-type.partial` background rgba and color.
  - `#059669` and `#2563EB` in dark mode overrides for `.create-btn.full-backup` and `.partial-backup` (line ~1150).
  - `rgba(59,130,246,0.04)` and similar rgba hardcoded in hover backgrounds and selected states.
  - `rgba(76, 175, 80, 0.15)` and `rgba(239, 68, 68, 0.15)` for badges.
  - `rgba(33, 150, 243, 0.1)` and `rgba(33, 150, 243, 0.04)` used repeatedly for backgrounds.
  - `#64748b` and similar hex colors used in inline styles for text color fallback.
  - `#e2e8f0` and `#f8fafc` used as fallback colors in inline styles.

- These hardcoded colors mostly correspond to Bento palette colors or their rgba variants, but are not referenced via tokens.

---

### 3. DARK MODE

- Has explicit `@media (prefers-color-scheme: dark)` block starting line ~1150.
- Inside dark mode, overrides Bento tokens with darker variants for background, card, border, text, shadows.
- Also sets `color-scheme: dark !important;` inside dark mode.
- Uses Bento tokens with dark variants in dark mode.
- The `:host` sets `--dark-mode` CSS variable based on `this._hass?.themes?.darkMode` (line ~1020).
- Conclusion: Dark mode support is implemented correctly with Bento tokens and overrides.

---

### 4. RESPONSIVE

- Media queries present:

  - `@media (max-width: 768px)` (lines ~1100 and ~1200) for card padding, grid columns, flex direction.
  - `@media (max-width: 480px)` (line ~1110) for tabs gap, tab button padding/font size, grid columns.
  - `@media (max-width: 600px)` (line ~1050) for backup-header flex direction, health-grid and contents-grid columns.
  - No `360px` breakpoint found.

- Content that might overflow on mobile:

  - Tabs have `overflow-x: auto` and `-webkit-overflow-scrolling: touch` to handle overflow.
  - Tables have fixed widths and `table-layout: fixed` (line ~1040), which helps prevent overflow.
  - `.compact-name` and `.compact-location` cells use `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;` to prevent overflow.
  - Overall, responsive design is handled well with grid and flex adjustments and scrollable tabs.

### 6. FONT

### 8. ACCESSIBILITY

- Color contrast:

  - Uses Bento tokens which are designed for good contrast.
  - Warning color (#F59E0B) used for alerts and warnings, with sufficient contrast on white backgrounds.
  - Error color (#EF4444) used for error banners with white text.
  - Text colors use Bento text and secondary text tokens for readability.

- Aria labels:

  - Buttons and tabs lack explicit `aria-label` or roles.
  - Tabs are buttons with `data-tab` attributes but no ARIA roles like `tablist`, `tab`, `tabpanel`.
  - No keyboard navigation hints or focus outlines explicitly defined, but buttons have default focus styles.
  - Table headers are present, but no `scope` attributes on `<th>`.
  - No explicit aria-live regions for error banners or dynamic content.

- Suggestion: Add ARIA roles and labels for tabs and interactive elements to improve accessibility.
