- The layout uses flexible CSS like flexbox and grid with wrapping (e.g. `.device-tabs` uses `flex-wrap: wrap`).
- Some elements have `min-width` and `max-width` constraints (e.g. inputs in settings tab).
- The main card width is not explicitly constrained, but Bento cards usually adapt to container width.
- Potential overflow risk:

  - Some inline styles use fixed widths (e.g. battery bar width 110px SVG), but this is small enough.

- Verdict: No explicit responsive breakpoints, but layout is mostly flexible and scrollable. No obvious overflow issues on mobile.

  - Text colors use Bento tokens which are designed for accessibility.
  - Some inline text colors like `#6b7280` on white backgrounds have acceptable contrast.
  - `.alert-banner` uses red backgrounds with lighter text color (#fca5a5), which might be borderline but acceptable given background color.

- Aria labels:

  - The tip banner dismiss button has `aria-label="Dismiss"` (line ~1080), good for screen readers.
  - Other interactive elements (buttons) do not have explicit aria-labels but have clear text content or tooltips (`title` attributes on maintenance buttons).
  - The refill button has clear text content and icon.
  - Tabs and device tabs are buttons with text labels.
  - No explicit roles or aria-selected on tabs, which could be improved for screen reader users.
  - Color is not the only indicator of state (icons and text labels present).

- Keyboard accessibility:

  - Buttons are native `<button>` elements, so keyboard focus and activation should work.
  - No visible focus styles defined explicitly, but Bento tokens and browser defaults likely apply.

- Verdict: Mostly accessible with good color contrast and labels; minor improvements possible on ARIA roles for tabs.
